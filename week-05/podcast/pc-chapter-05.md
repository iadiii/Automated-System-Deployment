# Podcast Questions - Chapter 05

[Command Line Heroes: Bash](https://www.redhat.com/en/command-line-heroes/season-3/heroes-in-a-bash-shell "Command Line Heroes: bash")

* ~0:20 Who is the creator of the Bash Shell?
  * Brian Fox
* ~0:43 Which organization did the creator of the Bash Shell write the shell for?  
  * Brian Fox, the creator of the Bash Shell, wrote it for Free Software Foundation.
* ~2:05 How does the podcast host define a shell script?
  * Podcast host defines shell script as a script given to an actor. Just like a shell delivers a whole sequence of commands, the same way an actor can read her lines one after another.
* ~2:23 Shell scripts are the key to what?
  * They are a key to automation.
* ~3:28 When did Ken Thompson release his shell and what was it missing?
  * It was released in 1971. It was missing on full-on scripting.
* ~3:45 What year did shell scripting come into existence?
  * It came into existence in 1977
* ~4:27 What was the shell that became the AT&T UNIX standard shell?
  * The Bourne Shell
* ~5:53 The Bourne Shell was licensed and owned by whom?
  * It was owned by Bell Labs and was licensed under UNIX.
* ~7:59 Why was Brian Fox the perfect person to develop the Bash Shell?
  * He knew code better than most of the folks at Bell Labs. Also, he had never been in  a position to see any of the source code that made the Bourne shell work, and that made him ideal for the task at hand.
* ~9:30 How long did it take to create the Bash shell and what was difficult about this?
  * It took nine months. The difficult part of this shell was to mimic the original Bourne shell, but at the same time to make it a better tool for people to use.
* ~12:02 What did Brian accidentally do to the Bash Shell code?
  * He had created a file in his directory called asterisk dot C, and he thought, he should get rid of that file, so he typed in RM space, quote, star dot C, close quote, which in a modern shell, when we use the quotes, it means, "Do not expand this" and then he pressed return, and it took a long time to come for the prompt. Then he realised that he deleted all the files from the directory.
* ~14:48 What was the other shell released one month before Bash?
  * ASH an open source shell.
* ~15:19 When was GNU Bash released?
  * In 1989
* ~18:40 What was the released/intended purpose of GNU Bash?
  * So that it was backwards compatible.
* ~19:25 What words and terms are now in use in everyday English?
  * Open Source and free software
* ~20:46 Was Steven Bourne "cool" with the Bash Shell?
  * Yes he was cool with it, there is no bitterness at all.
* ~22:52 What prepares you to be more of a long-term thinker?
  * Learning scripting prepares us to be a long term sort of thinker.
